# Evolution of Todo - Hackathon II Constitution

## Quick Reference

- **Project**: Evolution of Todo
- **Hackathon**: Panaversity Hackathon II
- **Constitution Version**: 1.0.0
- **Ratified**: 2026-01-02

## Core Principles

1. **Spec-Driven Development (MANDATORY)** - Every feature starts with a written specification
2. **Zero Manual Coding** - Code only generated by Claude Code, no manual edits
3. **Phase-by-Phase Progression** - 5 sequential phases, no skipping
4. **Reusable Intelligence Priority** - Use agent skills and subagents
5. **Strict Tech Stack Compliance** - Only use specified technologies per phase
6. **Security & Best Practices** - User isolation, stateless design, clean code
7. **Monorepo with Phase Isolation** - Each phase in isolated folder
8. **Documentation Excellence** - README, CLAUDE.md, spec history required

## Required Features

### Basic (All Phases)
- Add Task (title + optional description)
- Delete Task
- Update Task
- View Task List
- Mark as Complete / Incomplete

### Intermediate (Phase V)
- Priorities (high/medium/low)
- Tags/Categories
- Search by keyword
- Filter by status, priority, date
- Sort by due date, priority, title

### Advanced (Phase V)
- Recurring Tasks
- Due Dates & Time Reminders

### AI Integration (Phases III–V)
- Natural language interface

## Workflow

Specify → Plan → Tasks → Implement

**Rule**: No code generation without Task ID reference.

## Tech Stack Per Phase

- **Phase I**: Pure Python
- **Phase II**: Next.js + FastAPI + Neon + Better Auth (JWT)
- **Phase III**: OpenAI ChatKit + Agents SDK + Official MCP SDK
- **Phase IV**: Docker + Minikube + Helm + kubectl-ai/kagent
- **Phase V**: Dapr + Kafka/Redpanda + Cloud K8s
