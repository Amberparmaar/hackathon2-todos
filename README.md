# Evolution of Todo - Hackathon II

Welcome to the Evolution of Todo project! This is a comprehensive hackathon project organized by Panaversity that demonstrates the evolution of a simple todo application through 5 distinct phases, from an in-memory Python console app to a fully cloud-native, event-driven, AI-powered distributed system.

## ğŸš€ Project Overview

The Evolution of Todo project showcases a systematic approach to software development using **Spec-Driven Development (SDD)**. The project evolves through 5 sequential phases, each building upon the previous one to introduce new technologies, architectural patterns, and capabilities.

### Core Principles

- **Spec-Driven Development**: Every feature begins with a written specification
- **Zero Manual Coding**: Code is generated by Claude Code based on specifications
- **Phase-by-Phase Progression**: Each phase builds on the previous one
- **Reusable Intelligence**: Emphasis on creating reusable patterns and skills
- **Strict Tech Stack Compliance**: Following prescribed technology stacks for each phase
- **Security & Best Practices**: User isolation, stateless design, error handling
- **Monorepo with Phase Isolation**: Clear separation between phases
- **Documentation Excellence**: Comprehensive documentation throughout

## ğŸ“‹ Project Phases

### Phase I: In-Memory Python Console App
- **Technology**: Pure Python
- **Storage**: In-memory (no database)
- **UI**: Console-based
- **Constraints**: No external dependencies beyond stdlib

### Phase II: Full-Stack Web App
- **Technology**: Next.js (App Router + Tailwind), FastAPI, Neon PostgreSQL, Better Auth (JWT)
- **Features**: Basic CRUD + User Authentication
- **Constraints**: Use only specified libraries

### Phase III: AI-Powered Chatbot
- **Technology**: OpenAI ChatKit UI, FastAPI, Cohere API, MCP-style tools
- **Features**: Natural Language Processing + Task Management
- **Constraints**: Stateless server with database-persisted conversations

### Phase IV: Local Kubernetes Deployment
- **Technology**: Docker, Minikube, Helm, kubectl-ai/kagent
- **Features**: Containerized deployment
- **Constraints**: Use Gordon for Dockerfiles if available

### Phase V: Advanced Cloud Deployment
- **Technology**: Dapr, Kafka/Redpanda, Cloud K8s
- **Features**: Event-driven architecture + all advanced features
- **Constraints**: Cloud K8s deployment only

## ğŸ”§ Getting Started

### Prerequisites
- Node.js 18+
- Python 3.9+
- PostgreSQL (optional, SQLite used by default)

### Phase II Setup

#### Backend
1. Navigate to `phase2/backend/`
2. Install Python dependencies: `pip install -r requirements.txt`
3. Set up environment variables (copy `.env.example` to `.env`)
4. Start the server: `python main.py` or `uvicorn server:app --reload`

#### Frontend
1. Navigate to `phase2/frontend/`
2. Install dependencies: `npm install`
3. Set up environment variables (copy `.env.example` to `.env`)
4. Start development server: `npm run dev`
